
# 춘천 신호등 정보 파악 PWA

## 핵심 원리: 반복되는 패턴
신호등은 고장 나지 않는 한 [초록불 시간 + 빨간불 시간]이라는 일정한 주기를 무한히 반복

나머지 연산: 현재 흐른 시간을 120초로 나눴을 때 남는 나머지 숫자가 30보다 작으면 초록불, 크면 빨간불(노란불도 빨간불에 포함)로 판단하게 시켰습니다.

## 코드의 주요 구성 요소
동기화 버튼: 기준 잡기
신호등이 언제 바뀌었는지 모르기 때문에 사용자가 실제 신호등을 보고 초록불 버튼을 눌러 기준 시간(startTime)을 알려주는 것입니다.

로컬 스토리지(LocalStorage): 정보 기억
브라우저를 껐다가 다시 켜도 아까 설정한 기준 시간을 기억하게 해줍니다. 덕분에 한 번만 제대로 맞춰두면 내일 출근할 때 앱을 켜자마자 바로 남은 시간을 볼 수 있습니다.

실시간 업데이트(setInterval)
0.5초마다 현재 시각과 기준 시각을 비교하여 화면의 숫자와 색깔을 자동으로 바꿉니다.

## 사용 시나리오
사전 준비: 이번 주 출근길에 신호등 시간을 재봅니다.

설정 입력: 웹 페이지 하단 설정창에 초록불 시간과 빨간불 시간을 입력합니다.

동기화: 신호등 앞에 서서 초록불이 켜지는 순간 초록불 버튼을 누릅니다.

확인: 신호등 근처에 왔을 때 앱을 켜면 빨간불 10초 남음 같은 정보를 미리 알 수 있습니다.

## 한계
오차 발생: 스마트폰의 시계와 실제 신호등 시계가 미세하게 다를 수 있어 하루에 1~2초씩 오차가 생길 수 있습니다. 그럴 땐 다시 한번 동기화 버튼만 눌러주면 해결됩니다.
