
# 춘천 신호등 정보 파악 PWA

## 핵심 원리: 반복되는 패턴
신호등은 고장 나지 않는 한 [초록불 시간 + 빨간불 시간]이라는 일정한 주기를 무한히 반복

나머지 연산: 현재 흐른 시간을 120초로 나눴을 때 남는 나머지 숫자가 30보다 작으면 초록불, 크면 빨간불(노란불도 빨간불에 포함)로 판단하게 시켰습니다.

## 코드의 주요 구성 요소
1. 지능형 주기 측정 스톱워치
실제 신호등 주기를 몰라도 괜찮습니다. '측정 시작' 버튼을 누르고 신호가 바뀔 때마다 클릭하면, 앱이 자동으로 초록불과 빨간불의 길이를 계산하여 설정에 입력합니다.

2. 실시간 동기화 (Sync)
실제 신호등이 초록불로 바뀌는 찰나에 버튼을 누르면, 그 시점이 **'절대 기준점'**으로 기록됩니다. localStorage 기술을 사용하여 브라우저를 껐다 켜도 기준점은 유지됩니다.

3. 5초 전 시각적 경고 (Flash)
신호가 바뀌기 5초 전부터 화면 전체가 깜빡입니다. 스마트폰을 손에 들고 있지 않아도 시야 끝에 걸리는 화면 변화로 신호 변경을 미리 인지할 수 있습니다.

4. 실시간 시계 연동
스마트폰 시스템 시계와 연동되어 작동하므로, 한 번만 맞춰두면 다음 날 같은 시간에 앱을 켜도 별도의 조작 없이 신호 상태를 바로 확인할 수 있습니다.

## 사용 시나리오
측정: 신호등 앞에서 [주기 측정 시작]을 눌러 해당 신호등의 데이터를 확보합니다.

동기화: 실제 초록불이 시작될 때 버튼을 눌러 앱과 실제 신호의 싱크를 맞춥니다.

예측: 이제 신호등 근처에 도달하기 전, 앱을 켜서 "앞으로 10초 뒤에 초록불이 된다"는 사실을 미리 확인하고 발걸음을 조절합니다.

## 한계 및 주의사항
물리적 오차: 기계적 결함이나 온도 변화 등으로 인해 실제 신호등 제어기와 스마트폰 시계 사이에 매일 약 1~2초 내외의 오차가 발생할 수 있습니다. 주기적으로 [동기화] 버튼을 눌러 보정해주는 것이 좋습니다.

가변 신호: 교통량에 따라 주기가 변하는 가변 신호등 구간에서는 예측이 부정확할 수 있습니다.

안전 최우선: 본 앱의 타이머 숫자는 보조적인 수단일 뿐입니다. 반드시 실제 신호를 눈으로 확인하고 건너세요!
